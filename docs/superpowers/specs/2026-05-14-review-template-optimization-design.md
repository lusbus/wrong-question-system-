# 错题复盘模板优化设计文档

## 概述

将现有的简单 AI 分析升级为结构化的五步复盘系统，帮助用户深度理解错题、提炼避坑口诀、建立错题关联网络，并提供总体优势和薄弱点的统计分析。

## 核心变更

### 1. 智能粘贴解析

**当前：** 用户手动填写题目内容、自己的答案，没有正确答案输入
**优化后：** 用户直接粘贴完整题目（含选项和正确答案），AI 自动识别提取

**实现要点：**
- 新增智能解析 Prompt，让 AI 从粘贴内容中提取：题目题干、各选项、正确答案
- 用户界面改为单一粘贴框 + 简单的答案选择（如 A/B/C/D）
- AI 自动判断用户选的对错

### 2. 五步结构化复盘

AI 自动生成以下五个部分的完整分析：

| 步骤 | 内容 | 说明 |
|------|------|------|
| 第一步 | 拆结构 | 将原文拆分成层次，标注每层作用（铺垫/过渡/重心），判断重心句位置 |
| 第二步 | 分析错因 | 从 8 种标准错因类型中选择最贴合的，并说明具体原因 |
| 第三步 | 对比选项 | 逐项分析每个选项，说明为什么对/为什么错，使用标准排除术语 |
| 第四步 | 提炼避坑口诀 | 用一句话总结这道题的教训 |
| 第五步 | 同类题标记 | 标记这道题和之前哪道题一样，记录自查动作 |

### 3. 总体优势和薄弱点分析

**新增功能：**
- 按错因类型统计（哪种错因最常犯）
- 按题型分布统计（哪个题型错误最多）
- 按知识点薄弱点统计
- 识别高频错误模式
- 生成个性化学习建议

## 数据模型变更

### WrongQuestion 表新增字段

```sql
-- 新增字段
structureAnalysis TEXT          -- 拆结构分析（JSON 格式存储层次信息）
errorCauseType TEXT             -- 错因类型（8 种标准类型之一）
errorCauseDetail TEXT           -- 具体错因说明
optionAnalysis TEXT             -- 选项对比分析（JSON 格式）
avoidPitfallMantra TEXT         -- 避坑口诀
similarQuestionIds TEXT         -- 同类题 ID 列表（JSON 数组）
selfCheckAction TEXT            -- 自查动作描述
```

### 新增统计视图

```sql
-- 按错因类型统计
CREATE VIEW error_cause_stats AS
SELECT errorCauseType, COUNT(*) as count
FROM wrong_questions
WHERE errorCauseType IS NOT NULL
GROUP BY errorCauseType;

-- 按题型统计
CREATE VIEW question_type_stats AS
SELECT questionType, COUNT(*) as count
FROM wrong_questions
GROUP BY questionType;
```

## 技术实现要点

### 1. Prompt 优化

新增三个 Prompt：
- `parseQuestion` - 智能解析粘贴的题目内容
- `analyzeReview` - 生成完整的五步复盘分析
- `generateStudyAdvice` - 基于统计数据生成学习建议

### 2. 数据库迁移

- 新增字段到 wrong_questions 表
- 新增统计视图
- 迁移现有数据（设置默认值）

### 3. 前端组件

- 修改 InputPage 为智能粘贴模式
- 新增 ReviewResult 组件展示五步复盘
- 新增 StatisticsPage 展示统计分析
- 修改 QuestionForm 支持新的数据结构

### 4. Markdown 模板

更新错题本 Markdown 模板，使用结构化格式输出五步复盘结果。

## 实施阶段

### 阶段一：核心复盘功能
1. 修改数据模型（新增字段）
2. 优化 AI Prompt（智能解析 + 五步复盘）
3. 修改前端录入和展示界面

### 阶段二：统计分析
1. 新增统计分析页面
2. 新增统计视图和 API
3. AI 学习建议生成功能

### 阶段三：Markdown 优化
1. 更新 Markdown 模板
2. 确保导出格式美观
3. 支持复盘卡单独导出

## 风险与注意事项

1. **AI 解析准确率** - 智能解析粘贴内容可能识别不准，需要用户确认机制
2. **Prompt 长度** - 五步复盘的 Prompt 较长，需要控制 token 消耗
3. **数据迁移** - 现有错题数据需要兼容处理
4. **性能** - 统计分析需要聚合查询，需要优化 SQL
